# Contributing

Thanks for your interest in contributing!

## Quick start

- Fork the repo and create a feature branch.
- Run `npm install` and `npm test` before submitting a PR.
- Keep changes focused and include tests for new behavior.

## Development workflow

1. `npm install`
2. `npm run build`
3. `npm test`

## Code style

- Use TypeScript strictness as configured.
- Prefer small, well-scoped commits.
- Avoid introducing non-ASCII characters unless required.

## Reporting issues

When filing issues, include:

- Steps to reproduce
- Expected vs. actual behavior
- Logs or screenshots if relevant
- Your environment (OS, Node version, x64dbg version)

## Security

Please report security issues privately. See `SECURITY.md`.
